# Умный торговый автомат

REST API для управления умным торговым автоматом: приём монет, продажа товаров, техническое обслуживание. 
Автомат нагревается со временем, товары теряют свежесть, а при перегреве автомат необратимо ломается.

## Стек

- Node.js
- TypeScript
- Express.js
- Хранение состояния: in-memory

## Установка и запуск

В терминале:
- npm install
- npm start
Сервер поднимается на http://localhost:3000

### Установка через Docker

В терминале:
- docker compose up
Для завершения работы
- docker compose down 

## Логика работы автомата
Автомат начинает работу со следующими стартовыми значениями полей:
- temperature = 20;
- credit = 0;
- revenue = 0;
- slots = [];
Структура интерфейса Slot:
- id;
- price;
- stock;
- freshness;
- product;
Автомат работает во всех состояних, кроме broken
Раз в минуту поле temperature увеличивается на 3 ед., а freshness у slots уменьшается на 1.
Если поле temperature > 100, то автомат необратимо ломается, что не даёт пользователю взаимодействовать с автоматом за исключением просмотра состояния.
Основная работа:
- Пользователь вносит credit, который сохраняется в автомате;
- Пользователь по id полю slot выбирает нужный ему товар;
- Автомат списывает 1 единицу выбранного slot, уменьшает credit на значение price, увеличивает revenue на значение price;
- Также есть возможность обновить slots в автомате. Метод restock заменяет все поля у slots, что прислал пользователь и увеличивает freshness до 100. 

## API
Все запросы, которые работают в проекте лежат в файле request.http

### GET /machine
Получить текущее состояние автомата.

**Ответ 200:**
{
  "machine": {
    "temperature": 20,
    "credit": 0,
    "revenue": 0,
    "status": "operational",
    "slots": []
  }
}

### POST /machine/insert
Внести монеты.

**Запрос:**
{
  "amount": 100
}

**Ответ 200:**
{
  "credit": 100
}

### POST /machine/restock
Заполнить или обновить товар в слоте. Свежесть сбрасывается в 100.

**Запрос:**
{
  "id": 1,
  "product": "Cola",
  "price": 130,
  "stock": 15
}

**Ответ 200:**
{
  "machine": {
    "temperature": 20,
    "credit": 0,
    "revenue": 0,
    "status": "operational",
    "slots": [
      {
        "id": 1,
        "price": 130,
        "stock": 15,
        "product": "Cola",
        "freshness": 100
      }
    ]
  }
}

### POST /machine/select
Купить товар из слота.

**Запрос:**
{
  "slotId": 1
}

**Ответ 200:**
{
  "result": {
    "credit": 70,
    "slot": {
      "id": 1,
      "price": 130,
      "stock": 14,
      "product": "Cola",
      "freshness": 100
    }
  }
}

### POST /machine/maintain
Провести обслуживание (−30 к температуре, не ниже 0).

**Ответ 200:**
{
  "result": {
    "temperature": 0,
    "status": "operational"
  }
}

## Обработка ошибок

| Код | Ситуация |
|-----|----------|
| 400 | Некорректные данные, отсутствуют поля, неверный тип, битый JSON |
| 402 | Недостаточно кредита |
| 404 | Слот не найден |
| 409 | Автомат сломан / товар закончился / товар испорчен |
| 500 | Непредвиденная ошибка сервера |

**Примеры ошибок:**
{
  "message": "Insufficient credit. Required: 130, Current: 70."
}

{
  "message": "slotId must be a number",
  "field": "slotId"
}

## Структура проекта

src/
├── domain/        <- бизнес-логика (Machine, Slot)
├── errors/        <- иерархия доменных ошибок
├── middleware/    <- логирование, обработка ошибок
├── routes/        <- HTTP-эндпоинты и валидация
├── state.ts       <- единственный экземпляр автомата
└── index.ts       <- сборка приложения, запуск, таймер
